# 面试讲解稿

## 1 分钟介绍

这个项目是一个 PromptOps 评测与 Prompt 优化平台。它不是简单地调用一次大模型，而是围绕 Prompt 的版本管理、测试集管理、批量运行、自动评分、人工复核、失败案例分析和版本对比，做了一条完整的 Prompt 工程化链路。

项目默认使用 Mock 模型，所以没有 API Key 也能运行和演示。这样做的好处是稳定、可复现，也方便 HR 和面试官本地查看。

## 我解决的问题

真实的 Prompt 工作不是写一句提示词就结束，而是要回答这些问题：

- 这个 Prompt 在多种测试问题上是否稳定？
- 新版本 Prompt 是否比旧版本更好？
- 哪些问题失败了，为什么失败？
- 自动评分和人工评分是否能结合起来？
- 成本和耗时是否可控？

这个项目就是围绕这些问题设计的。

## 技术实现

后端使用 FastAPI，数据存 SQLite。核心逻辑拆成几个服务：

- `prompt_renderer`：把测试问题填入 Prompt 模板。
- `mock_model`：生成稳定的模拟模型输出。
- `evaluator`：根据期望关键词计算自动评分。
- `batch_runner`：批量运行测试集并汇总指标。
- `repository`：保存 Prompt、测试用例、运行记录、输出、评分和失败案例。

前端用原生 HTML/CSS/JS，直接调用 FastAPI 接口。页面能管理 Prompt 版本和测试用例，也能运行测试、人工评分、查看失败案例和版本对比。

## 亮点

- 支持 Mock 模式，不依赖真实模型 API。
- Prompt 版本对比不是静态展示，而是根据真实运行历史聚合出来。
- 失败案例不是手写假数据，而是由评分失败自动产生。
- 支持人工评分，模拟真实 Prompt 评测中的人工复核流程。
- 有一键重置演示数据，适合录屏和现场演示。

## 可以继续扩展的方向

- 接入真实模型 API，例如 OpenAI、DeepSeek、Claude。
- 增加 LLM-as-Judge 评分。
- 增加测试集导入导出。
- 增加多维评分维度，例如准确性、完整性、格式、语气。
- 增加用户登录和项目空间。
