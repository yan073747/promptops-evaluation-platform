# 技术讲解

## 1 分钟项目介绍

这是一个 PromptOps 评测与 Prompt 优化平台，用来管理 Prompt 模板、测试集、模型输出、评分结果、失败案例和优化历史。

项目默认使用 Mock 模型，所以没有 API Key 也能运行和演示。这样做的好处是稳定、可复现，也方便在任何本地环境查看完整流程。

平台模拟的是一个真实 Prompt 工程流程：先维护 Prompt 版本和测试用例，再批量运行测试，之后系统自动评分，同时支持人工复核。对于低分输出，系统会生成失败案例，最后用版本对比看不同 Prompt 的效果差异。

## 解决的问题

很多 AI 应用只关注“能不能生成回答”，但工程上更重要的是“回答质量是否稳定”。如果没有测试集、评分和版本对比，就很难判断 Prompt 修改之后到底有没有变好。

这个项目用一个小型平台把这些环节串起来：

```text
Prompt 版本 -> 测试集 -> 批量运行 -> 输出记录 -> 自动评分 -> 人工复核 -> 失败分析 -> 版本对比
```

## 技术实现

- FastAPI 负责提供 API 和托管静态前端页面。
- SQLite 负责保存 Prompt 版本、测试用例、运行记录、模型输出、评分和失败案例。
- Mock 模型负责生成稳定输出，保证项目不依赖外部 API。
- 自动评分器通过期望关键词命中率计算得分。
- 失败分析根据评分结果生成失败原因、严重程度和优化建议。
- 前端通过 `fetch` 调用真实 API，不是写死的静态页面。

## 工程亮点

- Mock first：没有 API Key 也能完整运行。
- 数据闭环：从 Prompt、测试集、输出、评分到失败分析都有记录。
- 版本对比真实化：不是写死指标，而是从历史运行记录聚合。
- 人工评分可追溯：每条模型输出都可以补充人工分数和复核备注。
- 一键重置：便于反复演示和调试。

## 后续扩展方向

- 接入真实模型供应商，例如 OpenAI、Claude、Gemini、DeepSeek。
- 增加语义相似度评分，而不仅是关键词评分。
- 增加 Prompt 优化建议生成。
- 增加用户登录和项目空间。
- 增加更多图表，比如版本趋势、失败类型分布、成本趋势。
